Module 5 Progress Report

On this Module 5 i will learn the following

Public Key Infrastructure is a framework used to securely manage encryption keys and digital certificates. It enables secure communication over the internet by using asymmetric encryption, where a public key encrypts data and a private key decrypts it. Key components of PKI include:

  • Certificate Authorities (CAs): Trusted organizations that issue and verify digital certificates.
  • Registration Authorities (RAs): Entities that authenticate the identity of users before certificates are issued.
  • Digital Certificates: Electronic credentials that link a public key with an entity's identity (like a domain or person).
  • Key Pairs: The public and private keys used in asymmetric encryption.

Cryptographic protocols are standardized methods that use encryption to protect data during communication. They define how algorithms and keys are used to achieve security goals like confidentiality, integrity, and authentication. Common cryptographic protocols include:

  • TLS (Transport Layer Security): Secures internet traffic (e.g., HTTPS).
  • IPSec: Secures network traffic at the IP layer, used in VPNs.
  • SSH (Secure Shell): Encrypts remote login sessions.
  • PGP (Pretty Good Privacy): Encrypts emails and files.
  • S/MIME: Secures email communication using certificates.
